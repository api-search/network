  The GET Views transactions API allows you to fetch all the transactions for a specific view by passing the viewId.<br><br> A View is a logical collection of several rules. This API fetches all the transactions that qualify the rules associated with a specific view.<br><br> This API supports pagination. Each page can contain a maximum of 500 transactions. Use 'skip' and 'top' fields to enable pagination in response.<br><br> Supported containers - bank and card
